// how it works
Zero-config fleet monitoring

Three components, one command to enroll. Works on Linux, macOS, and Windows.

01 / deploy

Deploy the cloud brain

One-click Vercel deploy. Point it at a MongoDB database and you're done. Serves the dashboard and accepts agent connections.

vercel deploy ./cloud
02 / install

Install the agent

Global npm package. Runs setup wizard to choose project folders, configure startup, and enroll with your cloud instance.

npm i -g @wahid7852/watchd-agent
watchd setup
03 / watch

Open the dashboard

Your machine appears in the fleet, packages are indexed, vulnerabilities are matched against OSV. Live heartbeats keep status current.

https://your-cloud.vercel.app
// features
Built for security-conscious teams

Every feature ships by default. Nothing to configure beyond the initial setup.

🔍

Passive project discovery

The agent scans watched folders for package.json and requirements.txt files. No app-level integration required.

⚡

OSV vulnerability matching

Package versions are checked against the Open Source Vulnerability database. Semver range matching catches indirect exposure.

🖥️

Multi-machine fleet view

Every enrolled machine reports to a single dashboard. See cross-fleet exposure at a glance, drill into any machine instantly.

🔄

Auto-remediation queue

One click sends a patch command back to the agent. The agent updates the package and reports the new version automatically.
